# Nova Realtime Contract (v1)

This document defines the websocket protocol shared by `nova-frontend` and
`nova-live-tutor`.

## Endpoint

- `ws://<host>/ws` (or `wss://` in production)
- Frontend default: `NEXT_PUBLIC_WS_URL`
- Requires query param: `token=<supabase_access_token>`

## Authentication

- Backend verifies `token` with Supabase Auth API: `GET /auth/v1/user`
- Missing token -> websocket `error: missing_access_token` and close (4401)
- Invalid token -> websocket `error: invalid_access_token` and close (4401)
- Missing backend Supabase config -> `error: supabase_config_missing`

## Event Envelope

All events must use the JSON shape:

```json
{ "type": "<event_name>", "data": "<string_payload>" }
```

`data` is optional only for `ping` sent by the client.

## Client -> Server Events

- `audio`: base64 encoded PCM chunk
- `video`: base64 encoded JPEG frame
- `ping`: connectivity check

## Server -> Client Events

- `status`: lifecycle + health updates (`connected`, `pong`, etc.)
- `audio`: base64 encoded PCM chunk generated by tutor
- `error`: protocol or payload validation errors

## Validation Rules

- Invalid JSON -> `error: invalid_json`
- Non-object event -> `error: invalid_event_shape`
- Unknown event type -> `error: unsupported_event_type`
- Non-string payload for `audio`/`video` -> `error: invalid_data_payload`
- Invalid base64 audio payload -> `error: invalid_audio_payload`
- Supabase auth unavailable -> `error: auth_service_unavailable`

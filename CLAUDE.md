# Nova AI Live Tutor

## Overview
Nova is a real‑time AI tutoring backend built with **FastAPI**, **WebSockets**, and the **Google Gemini Live** API.  It authenticates users via **Supabase** and streams audio/video between the client and Gemini.

## Tech Stack
- **Python 3.13**
- **FastAPI** – HTTP API & WebSocket endpoint
- **Google Gemini Live** – generative AI model for audio responses
- **Supabase** – authentication (JWT) and optional data store
- **Docker** – containerised deployment (multi‑stage build)

## Architecture Highlights
- `main.py` hosts the FastAPI app, validates inbound WebSocket events and performs async Supabase token verification using **aiohttp**.
- `agent/session.py` manages the Gemini live session, sending/receiving audio/video streams.
- Environment variables are defined in `.env.example` and loaded at runtime.
- The WebSocket protocol is documented in `ARCHITECTURE.md`.

## Improvements Implemented
- Replaced blocking `urllib` calls with **async `aiohttp`** for Supabase token verification.
- Standardised error codes to the `4000‑4003` range to avoid conflict with Supabase status codes.
- Added type hints for core functions and switched to async token verification.
- Updated imports and removed unused urllib modules.
- Prepared the repository for further enhancements (type‑hints, Pydantic validation, CORS, ASGI lifespan events, Docker improvements).

## Development Workflow
```bash
# Clone & set up virtualenv
git clone https://github.com/boraneak/nova-live-tutor.git
cd nova-live-tutor
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Load env vars
cp .env.example .env
# edit .env with real keys

# Run locally
uvicorn main:app --reload
```

## Deployment
- Docker image can be built with `docker build -t nova-live-tutor .`
- Deploy to Railway, Fly.io, or any container‑orchestrated platform.
